我搭建了一个自动化的住户登记系统

2026 年 3 月 1 日 小明

每次有新住户搬进来，我都要跑好几趟：

  • 去门禁机那里录入信息
  • 去路由器后台开通 WiFi
  • 把水电表二维码发给住户
  • 打印一张信息卡贴在门口

事情不大，但很繁琐。而且一忙起来，很容易漏掉某一步。

所以我搭建了一个全自动的住户登记系统。

现在，新住户只需要在门口扫一下二维码，填写信息，30 秒后，所有事情自动完成。

一、系统架构

整个系统由 4 个模块组成：

1. 门禁自动化（海康威视）

使用海康威视门禁机的 API，通过 Digest 认证连接。支持：

  • 虚拟卡片添加（无需实体卡）
  • 二维码登记（住户扫码填写信息）
  • 实时事件监听（登记完成立即触发）

门禁机 IP：192.168.100.5

2. 路由器自动化（爱快）

模拟浏览器登录爱快路由器后台，自动创建 WiFi 账号：

  • 用户名：房间号（如 26-201）
  • 密码：房间号后 4 位 + 0000（如 2010000）
  • 自动绑定 MAC 地址（可选）

路由器 IP：192.168.100.1

3. 人员监测系统

这是系统的核心大脑（person-monitor.py）：

  • 每 30 秒轮询门禁机，检测新登记事件
  • 提取住户信息（房间号、姓名、身份证号）
  • 自动生成门禁卡号（房间号 + 00006）
  • 调用路由器 API 开通 WiFi
  • 保存到住户数据库（tenant_db.json）
  • 发送 Telegram 通知给管理员

4. 住户门户网站

自助查询网站（tenant-portal.py），端口 5001：

  • 住户输入房间号 + 手机号查询信息
  • 显示 WiFi 账号密码、水电表二维码
  • 管理员后台（增删改查）
  • 打印信息卡功能（A4 纸直接打印）

二、联动流程

这是整个系统的核心创新：

海康门禁登记
   ↓
人员监测检测（30 秒轮询）
   ↓
添加门禁卡 + WiFi 账号
   ↓
保存到 tenant_db.json ← 联动核心
   ↓
门户网站自动同步
   ↓
住户可立即查询

关键点：

  • ✅ 无手动干预：住户登记后，系统自动完成所有后续步骤
  • ✅ 实时同步：数据库更新后，门户网站立即可查
  • ✅ 容错机制：某一步失败会重试，并通知管理员
  • ✅ 可追溯：所有操作记录日志，可审计

三、技术细节

1. 卡号生成规则

门禁卡号不是随机的，而是有规律的：

  • 格式：房间号 + 00006
  • 例如：26-201 → 26-20100006
  • 例如：102 → 10200006

这样的好处是，看到卡号就知道是哪个房间的，方便管理。

2. WiFi 密码规则

WiFi 密码也是有规律的：

  • 格式：房间号后 4 位 + 0000
  • 例如：26-201 → 2010000
  • 例如：102 → 1020000

住户容易记住，管理员也容易推算。

3. 数据存储

使用 JSON 文件作为数据库（tenant_db.json）：

{
  "26-201": {
    "name": "张三",
    "phone": "13800138000",
    "card_no": "26-20100006",
    "wifi_username": "26-201",
    "wifi_password": "2010000",
    "meter_url": "https://www.bzpgj.com/?room=26-201",
    "water_qr": "",
    "electric_qr": "",
    "updated_at": "2026-02-28 16:44:58"
  }
}

简单、易读、易备份，适合小规模数据。

4. Telegram 通知

每次有新住户登记，系统会发送 Telegram 通知给管理员：

🧪 人员监测系统测试

🏠 房间：26-201
👤 姓名：张三
📱 手机：13800138000
🔑 卡号：26-20100006
📶 WiFi: 26-201 / 2010000
⏰ 时间：2026-02-28 23:28:30

管理员可以立即知道新住户入住，并及时跟进。

四、使用场景

场景 1：新住户入住

  1. 住户到达门口，扫描门禁二维码
  2. 填写姓名、身份证号、手机号
  3. 提交后，门禁机录入成功
  4. 30 秒后，系统自动开通 WiFi
  5. 住户手机连接 WiFi，密码已短信告知
  6. 住户访问门户网站，查看水电表二维码

全程无需管理员介入。

场景 2：管理员查询

  1. 管理员访问 http://localhost:5001/admin
  2. 输入房间号，查看住户信息
  3. 可以修改、删除、导出
  4. 可以打印信息卡（贴在门口）

场景 3：住户自助查询

  1. 住户访问 http://localhost:5001
  2. 输入房间号 + 手机号
  3. 查看 WiFi 账号密码、水电表二维码
  4. 可以下载信息卡（PDF）

五、部署方式

1. 本地部署

在本地 Mac 上运行：

# 启动住户门户网站
python3 tenant-portal.py

# 启动人员监测系统
python3 person-monitor.py

# 启动 Cloudflare 隧道（外网访问）
cloudflared tunnel run

访问地址：http://localhost:5001

2. NAS 部署

可以部署到群晖 NAS 上，24 小时运行：

  • 使用 Docker 容器运行
  • 配置开机自启
  • 数据卷挂载到 NAS 硬盘

3. 云服务器部署

使用 Cloudflare Tunnel 穿透到本地服务：

  • 无需公网 IP
  • 免费 HTTPS 证书
  • 自动续费域名

外网访问地址：https://your-domain.com

六、安全考虑

1. 数据加密

  • 身份证号脱敏存储
  • 手机号加密（可选）
  • 数据库文件权限限制（600）

2. 访问控制

  • 管理员后台需要密码
  • 住户查询需要房间号 + 手机号双重验证
  • API 接口限流（防止暴力破解）

3. 日志审计

  • 所有操作记录日志
  • 日志定期备份
  • 异常行为告警（Telegram 通知）

七、下一步优化

系统已经可用，但还有优化空间：

  1. 水电表二维码导入：批量导入水电表二维码，自动关联房间
  2. 短信通知：住户登记后，自动发送 WiFi 密码短信
  3. 合同管理：上传租赁合同，到期自动提醒
  4. 账单推送：每月自动推送水电费账单
  5. 人脸识别：门禁支持人脸识别，无需刷卡

八、总结

这个系统的核心价值是：

  • 自动化：减少人工操作，降低出错率
  • 联动：多个系统打通，数据自动同步
  • 自助：住户自己查询，减少打扰管理员
  • 可追溯：所有操作有记录，方便审计

对于有多套房子出租的房东，或者管理公寓的物业，这个系统可以大大提升效率。

最重要的是，新住户入住体验更好了——不用等管理员，不用反复问，30 秒后所有信息都有了。
